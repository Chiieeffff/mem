# LinkedIn Post Drafting — Workflow

## Objective

Turn a rough thought, topic, or observation into a ready-to-post LinkedIn post in Roman's voice. Output is text ready to copy-paste directly into LinkedIn — no editing required.

---

## Required inputs

- **Core idea**: 1–3 sentences describing the topic, thought, or experience
- **Optional — angle preference**: story-led (personal experience) OR insight-led (observation/opinion)
- **Optional — CTA preference**: direct ask / soft ask / no CTA

If the core idea is too vague to write from, ask one clarifying question before drafting.

---

## Step 1 — Determine the angle

If the user didn't specify:

| Use story-led when | Use insight-led when |
|---|---|
| The idea comes from a personal experience | The idea is an observation about an industry pattern |
| There's a concrete before/after | The idea challenges a common assumption |
| The reader benefits from following the narrative | The reader benefits from the conclusion immediately |

---

## Step 2 — Write the post

### Structure

**Hook line** (line 1 — stands alone before "see more")
- The only job of line 1 is to make someone stop scrolling
- Use a specific, counterintuitive, or tension-creating statement
- Never start with "I'm excited to share", "Thrilled to announce", or any hype opener
- No questions as hooks — they're overused and weak
- Target length: 8–15 words

**Body** (lines 3–10)
- Story-led: setup → what happened → what it revealed
- Insight-led: claim → supporting evidence or example → implication
- One idea per line where possible — LinkedIn renders line breaks, use them
- Concrete over abstract: numbers, names, specific outcomes over vague claims
- No bullet lists unless the content genuinely requires enumeration (3 items max if used)

**Closing / CTA** (lines 11–13)
- If direct ask: one clear action (reply, DM, share)
- If soft ask: a question that invites response without demanding it
- If no CTA: end on the sharpest line from the body — don't trail off

---

## Step 3 — Output format

If the user asked for one variant:
```
[Post text — ready to copy-paste]

---
Note: [1 sentence on the angle chosen and why, only if non-obvious]
```

If the user asked for two variants or didn't specify:
```
**Variant A — [Story-led / Insight-led]**
[Post text]

---

**Variant B — [Story-led / Insight-led]**
[Post text]

---

**Recommendation:** [A or B] — [one sentence on why]
```

---

## Voice rules (always apply)

- Write as Roman — first person, direct, no performative humility
- No emoji unless Roman specifically requests them
- No hashtag spam — 0–2 hashtags max, only if genuinely relevant
- No corporate language: no "leveraging", "synergies", "exciting journey", "thought leadership"
- Short sentences beat long ones
- If a sentence doesn't add something, cut it
- GatewayCrypto-related posts: can mention the product but should not read like an ad. Make the insight the point, not the plug.

---

## Length targets

| Post type | Target length |
|---|---|
| Insight / opinion | 80–150 words |
| Story | 120–200 words |
| Announcement | 60–100 words |

Never exceed 220 words. LinkedIn's algorithm does not reward length.

---

## Anti-patterns

- "I'm humbled by..." — cut
- "This one hit different" — cut
- Listicles with 7+ items
- Ending with "What do you think?" as the only CTA — it's lazy; ask something specific
- Padding the hook to make it dramatic when the idea is simple — just say the thing
- Starting with the company name or product name — lead with the human insight
